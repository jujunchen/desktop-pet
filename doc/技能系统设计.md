# 桌面宠物技能系统集成计划

## 上下文

### 问题
当前桌面宠物只有固定的内置工具（获取时间、系统状态、打开应用、执行命令等），缺少动态扩展能力。用户希望能够从 https://skills.sh/ 生态系统安装技能，让宠物获得新的能力。

### 现有架构
- **后端**: Rust + Tauri 2.0，已有完善的 `Tool` trait 和 `ToolRegistry` 系统
- **前端**: Vue 3 + TypeScript，聊天界面与 LLM 交互
- **LLM 代理**: ReAct 模式，支持工具调用
- **配置存储**: `~/.config/desktop-pet/config.json`

### Skills.sh 技能格式
- 技能存储在 `./.agents/skills/`（项目级）或全局目录
- 每个技能包含 `SKILL.md`，带有 frontmatter 元数据（name, description, metadata）
- 技能内容为 Markdown 格式的指令/提示，告诉 LLM 如何执行任务
- 部分技能包含可执行资源（bash 脚本等）

---

## 实现方案

### 核心设计理念
将 Skills.sh 技能作为 **LLM 能力增强层**：
1. 技能的描述和使用说明注入到 LLM 系统提示中
2. 对于包含可执行代码的技能，创建动态包装工具
3. 用户可以通过聊天界面或设置 UI 管理技能

## 详细实现步骤

### 阶段 1: 后端技能核心模块

#### 1.1 技能数据结构 (`src-tauri/src/skills/mod.rs`)
```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Skill {
    pub name: String,
    pub description: String,
    pub author: Option<String>,
    pub version: Option<String>,
    pub installed_at: i64,
    pub enabled: bool,
    pub skill_path: PathBuf,    // 技能安装路径
    pub content: String,         // SKILL.md 完整内容
    pub has_resources: bool,     // 是否有可执行资源文件
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SkillConfig {
    pub skills: Vec<Skill>,
    pub skills_dir: PathBuf,    // 技能存储目录（~/.config/desktop-pet/skills）
}
```

#### 1.2 技能管理器 (`src-tauri/src/skills/manager.rs`)
- `install_skill(package: &str) -> Result<Skill, String>`
  - 执行: `npx skills add <package> --skill '*' -y`
  - 解析安装后的 SKILL.md
  - 移动到应用配置目录
- `uninstall_skill(name: &str) -> Result<(), String>`
- `list_skills() -> Result<Vec<Skill>, String>`
- `enable_skill(name: &str) -> Result<(), String>`
- `disable_skill(name: &str) -> Result<(), String>`
- `search_skills(query: &str) -> Result<Vec<SkillInfo>, String>`
  - 执行: `npx skills find <query>`（需要处理交互式输出）

#### 1.3 SKILL.md 解析器 (`src-tauri/src/skills/parser.rs`)
- 解析 frontmatter（YAML 格式）
- 提取技能名称、描述、元数据
- 提取技能内容用于系统提示注入

#### 1.4 与现有 Tool 系统集成
- 修改 `ToolRegistry`，支持动态注册/注销技能工具
- 创建 `SkillTool` 包装器，为有资源的技能提供执行能力
- 修改 ReAct agent 系统提示构建，注入已启用技能的描述

### 阶段 4: LLM 集成

#### 4.1 修改 ReAct 系统提示构建
在 `src-tauri/src/llm/react.rs` 中：
```rust
// 在系统提示末尾注入已启用的技能描述
fn build_system_prompt(&self, registry: &ToolRegistry, skills: &[Skill]) -> String {
    let mut prompt = self.base_system_prompt.clone();
    
    // ... 现有工具描述 ...
    
    // 注入技能
    prompt.push_str("\n\n## 已启用的技能\n");
    for skill in skills {
        if skill.enabled {
            prompt.push_str(&format!("- **{}**: {}\n", skill.name, skill.description));
            prompt.push_str(&format!("  使用说明: {}\n\n", skill.content));
        }
    }
    
    prompt
}
```

## 安全考虑

1. **命令执行白名单**: `npx skills` 命令仅允许特定参数
2. **技能脚本审查**: 安装前显示技能安全评估（如 skills.sh 提供的风险等级）
3. **沙箱执行**: 技能脚本在受限环境中执行，避免危险操作
4. **权限提示**: 首次执行技能脚本时提示用户确认
