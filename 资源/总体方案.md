# AI桌面宠物助手 - 技术方案

## Context

用户需要开发一个基于AI的桌面宠物助手，具备以下核心功能：

1. 桌面宠物显示与智能动作（根据用户活动状态切换动作）
2. 支持鼠标点击互动，随机播放互动动作
3. 支持滚轮调整宠物大小，设置界面滑块精确调整
4. 可拖拽移动宠物到桌面任意位置
5. 可配置大语言模型Agent能力（OpenAI格式兼容）
6. 可配置ASR语音识别（本地Whisper / 阿里百炼 / 火山方舟 / FunASR私有化）
7. 右键隐藏功能（前台消失，后台驻留）
8. 右键完全退出功能
9. 技能系统扩展（可安装外部技能，LLM自动调用）

这是一个全新项目，需要从零开始构建。

## 技术栈选择

### 1. 桌面应用框架

**推荐：Tauri 2.0**

- ✅ 比Electron更轻量（包体积小10倍以上，内存占用低）
- ✅ Rust后端提供高性能，适合ASR本地推理
- ✅ 支持透明窗口、置顶窗口、无标题栏窗口
- ✅ 良好的系统托盘集成
- ✅ 跨平台支持（Windows/Mac/Linux）

**前端UI层：Vue 3 + TypeScript**

- 简单易用的动画系统
- 组件化开发，便于扩展宠物种类
- 成熟的生态系统

### 2. 宠物渲染与动画

**GIF动图方案（推荐）**

- 每个动作对应一个独立的GIF文件
- 实现简单，切换宠物只需更换GIF文件
- 素材容易获取（网上现成资源或AI生成）
- 透明背景GIF支持

**文件结构：**

```
src/assets/pets/dog/
├── resting.gif      # 趴着休息（用户无操作时）
├── sitting.gif      # 蹲着哈嘴（用户操作时）
├── lying-down.gif   # 趴下（指令触发）
├── talking.gif      # 说话/识别中
└── happy.gif        # 开心/回应时
```

**后期可升级选项：**

- Sprite精灵图（优化文件大小和性能）
- Lottie/Spine骨骼动画（需要美术资源）

### 3. 大语言模型集成

**多后端支持：**

- OpenAI API（gpt-4o-mini / gpt-4o）
- Anthropic Claude API
- 本地Ollama（支持Llama 3, Qwen等开源模型）
- 配置文件 `~/.config/desktop-pet/llm.json` 保存API Key和模型选择

### 4. ASR语音识别

**统一配置：所有在线服务复用同一套 OpenAI 格式配置结构**

| 提供商           | 类型  | 说明                                         |
| ------------- | --- | ------------------------------------------ |
| **本地Whisper** | 本地  | whisper.cpp，支持模型大小: tiny/base/small/medium |
| **阿里百炼**      | 在线  | OpenAI兼容格式，国内速度快                           |
| **火山方舟**      | 在线  | OpenAI兼容格式，火山引擎                            |
| **FunASR**    | 私有化 | 阿里开源，本地部署，完全免费                             |

**核心设计：统一Trait接口，新增提供商无需改前端代码**

### 5. 用户活动检测

**Rust系统级监听：**

- Windows: `user32.dll` 调用 `GetLastInputInfo`
- Mac: `IOKit` 或者 `CGEventSourceSecondsSinceLastEventType`
- Linux: `X11` 扩展 `XScreenSaverQueryInfo`
- 每5秒检测一次，超过30秒无输入判定为"用户离开"

## 核心功能实现方案

### 1. 宠物状态机（GIF实现）

```typescript
// 宠物状态
type PetState = 'resting' | 'sitting' | 'talking' | 'action'

// GIF映射表
const gifMap = {
  resting: '/pets/dog/resting.gif',   // 趴着休息
  sitting: '/pets/dog/sitting.gif',   // 蹲着哈嘴
  talking: '/pets/dog/talking.gif',   // 识别语音中
  action: '/pets/dog/lying-down.gif'  // 执行指令动作
}

// 状态切换规则：
// - 用户无操作 > 30秒 → resting（自动切到趴着GIF）
// - 用户活跃 → sitting（自动切到蹲着GIF）
// - 正在识别语音 → talking（切到说话GIF）
// - 收到指令 → action（播放对应动作GIF，完成后返回基础状态）
```

**Pet.vue 核心代码（超简单）：**

```vue
<template>
  <img 
    :src="currentGif" 
    alt="pet" 
    class="pet-image"
    @click="onPetClick"
    @mousedown="startDrag"
  />
</template>

<script setup>
const currentGif = computed(() => gifMap[petState.value])

// 点击互动
const onPetClick = () => {
  // 播放随机互动动作
  playTempAction(['happy', 'wag-tail', 'tilt-head'], { duration: 2000 })
}
```

### 互动动作系统

**动作优先级队列：**

```
临时互动动作 > 语音指令动作 > 基础状态动作

点击宠物 → 入队"happy"动作（2秒）→ 播放完 → 回到之前的状态
```

**GIF资源扩展：**

```
src/assets/pets/dog/
├── resting.gif          # 基础-趴着
├── sitting.gif          # 基础-蹲着
├── talking.gif          # 基础-说话中
├── happy.gif            # 互动-开心（点击触发）
├── wag-tail.gif         # 互动-摇尾巴
├── tilt-head.gif        # 互动-歪头杀
└── lying-down.gif       # 指令-趴下
```

### 2. 窗口配置

- 透明背景，无边框，始终置顶
- 右键菜单：显示/隐藏/设置/退出（系统托盘图标右键）
- 可拖拽移动宠物位置
- ✅ 支持鼠标点击互动
- ✅ 滚轮调整宠物大小（50% ~ 300%）
- ✅ 大小设置保存在配置文件

**点击穿透策略：**

- 整个宠物区域（非透明）接收鼠标事件
- 宠物图片外的透明区域：点击穿透，不影响操作其他窗口

### 3. 隐藏vs退出

- **隐藏**：调用 `appWindow.hide()`，后台进程继续运行，ASR和Agent保持监听
- **退出**：清理资源，调用 `app.exit()`

### 4. 语音对话流程

```
用户说话 → 麦克风录音 → Whisper识别文字 → LLM处理 → 
  1. 生成文本回复（可选TTS播报）
  2. 解析动作指令 → 触发对应宠物动画
```

### 5. 配置管理

**配置文件位置：`~/.config/desktop-pet/config.json`**

前端设置界面支持：

- **LLM配置**：API Key / Model / Base URL（代理/私有化用）
- **ASR配置**：4种提供商选择 + 对应参数（模型大小/密钥/服务地址）
- **宠物设置**：显示大小（滑块）、空闲进入休息时间
- 配置持久化，重启应用保持

## 验证方案

1. **基础功能测试**：启动应用，宠物显示在桌面，可拖拽移动
2. **状态切换测试**：
   - 不操作鼠标键盘30秒 → 宠物变为趴着状态
   - 移动鼠标 → 宠物立即变为蹲着状态
3. **互动测试**：点击宠物 → 播放开心/摇尾巴动作，2秒后自动回到基础状态
4. **菜单测试**：右键点击宠物，选择隐藏 → 宠物消失，进程仍在；重新显示后正常
5. **退出测试**：右键退出 → 进程完全终止
6. **ASR测试**：说"趴下" → 语音识别正确 → 宠物执行趴下动作
7. **LLM测试**：对话提问 → 得到AI回复
