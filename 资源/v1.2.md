# 宠物养成系统

## 原始需求

现在想要增加一个宠物养成系统，我的初步设想是用户第一次启动的时候，让用户进行一些初步配置，就是现在设置中的内容，额外增加一个模式的选择：宠物养成和宠物助手。

养成系统，在设置完成后，宠物开始只是一个幼体，功能是被限制的，只能逗他玩，给他喂食，能跟它聊天，但是不能给你执行指令，等到长大后，就能让他给你干活，能给你执行指令，养成系统宠物有一套属性：成长值，亲密度，规则是,每天挂着亲密度会按时间增加，逗它，喂食，都可以增加亲密度，亲密度达到一定的值后，成长值就会增长，这里设定宠物养一个星期后，即可长大，然后继续养的话，宠物会随着时间的增长而慢慢衰老，直到死亡，也就是跟真正的动物一样，这个时间一般根据正在动物生命时间来。

另一种模式，宠物助手，直接是一个成年的宠物，拥有完整的功能，不会死亡，成长值、亲密度也有，每天挂着亲密度会按时间增加，逗它，喂食，都可以增加亲密度，亲密度达到一定的值后，成长值就会增长。

## 一、核心数据结构设计

### 1.1 Rust 后端配置扩展 (`config.rs`)

在 `PetConfig` 中新增养成分支：

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct PetConfig {
    // 现有字段...
    #[serde(default = "default_pet_mode")]
    pub mode: PetMode,
    #[serde(default)]
    pub growth: GrowthState,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum PetMode {
    Growth,    // 养成模式
    Assistant, // 助手模式（直接成年）
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct GrowthState {
    pub stage: LifeStage,
    pub affection: f64,       // 亲密度 0-100
    pub growth: f64,          // 成长值 0-100
    pub hunger: f64,          // 饥饿值 0-100 (100=饱)
    pub happiness: f64,       // 快乐值 0-100
    pub health: f64,          // 生命值 0-100
    pub created_at: i64,      // 创建时间戳
    pub last_fed_at: i64,     // 上次喂食
    pub last_interacted_at: i64, // 上次互动
    pub reincarnation_count: u32, // 转世次数
    pub inherited_bonus: f64, // 继承加成（转世获得）
    pub last_updated_at: i64, // 上次状态更新时间
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum LifeStage {
    Baby,     // 幼体 (0-7天)
    Adult,    // 成年 (7天-10个月)
    Elder,    // 老年 (10个月-12个月)
    Dead,     // 死亡
}
```

## 三、核心逻辑设计

### 3.1 成长与生命周期计算

```
时间线：
0天 ────── 7天 ────────────── 300天 ───────── 365天
幼体         成年                  老年              死亡

成长值增长（所有数值随机）：
- 每天在线亲密度自然 +1~2
- 喂食：亲密度 +3~6，饥饿值 +30，快乐值 +10~15
- 玩耍（右键菜单）：亲密度 +1~2，快乐值 +3~5
  （随机播放摇头/开心/跳舞/接飞盘/说话动画，跟点击效果相同）
- 点击宠物：亲密度 +1~2，快乐值 +3~5（轻量互动）
- 聊天对话：亲密度 +1~3
- 成长值 = 亲密度（亲密度满100后继续累积推动阶段）

阶段转换：
- 幼体→成年：成长值 ≥ 100 或 ≥ 7天（满足任一即可）
- 成年→老年：≥ 300天
- 老年→死亡：≥ 365天 或 生命值归零
```

### 3.2 属性衰减逻辑（每分钟更新）

```
- 饥饿值: -0.5/分钟 (饿肚子会降低快乐值和生命值)
- 快乐值: -0.2/分钟 (饥饿会加速衰减)
- 生命值: 饥饿值<20 时 -0.3/分钟
- 亲密度: +0.1/分钟（在线时）
```

### 3.3 转世机制

```
死亡后选择转世：
- 继承上一世 10% 的亲密度和成长值作为加成
- 转世次数 +1
- 可以选择是否保留宠物名字
- 每转世一次，基础属性上限 +5%（最多 +50%）
```

### 3.4 功能权限控制

| 功能 | 幼体 | 成年 | 老年 | 死亡 |
|------|------|------|------|------|
| 聊天对话 | ✓ | ✓ | ✓ | ✗ |
| 执行系统指令 | ✗ | ✓ | ✓ | ✗ |
| 喂食/互动 | ✓ | ✓ | ✓ | ✗ |
| 右键菜单指令 | ✗ | ✓ | ✓ | ✗ |

## 五、关键实现要点

### 5.1 状态持久化
- 养成状态随配置一起保存到 `config.json`
- 每次状态更新后自动保存（防抖，最多每30秒存一次）
- 启动时恢复上次状态，并计算离线期间的属性变化

### 5.2 离线计算
- 启动时计算从 `last_updated_at` 到现在的时间差
- 根据离线时间批量更新属性值
- 离线超过24小时：宠物会很饿，生命值可能下降

### 5.3 事件驱动
- 使用现有 Tauri 事件系统：
  - `pet://growth-changed` - 属性变化通知
  - `pet://stage-changed` - 生命阶段变更通知
  - `pet://died` - 宠物死亡事件

### 5.4 与现有系统集成
- 复用现有配置加载/保存机制
- 复用宠物状态机和动作队列
- 复用右键菜单框架，新增菜单项
